Update: I have just realized that you can just clone the repository, and if you don't know, follow these steps:

  1. Open CMD (A.K.A Command prompt)
  2. And type in "git clone" with the repository URL
    3. git clone
  3. Then click enter.

And now, you have the repository on your computer, and since this is just a clone of the repository, so you can customize this clone repository as you like. Since it is not connected to the original repository. And btw here's the full git clone command:

git clone https://github.com/eltus/test-website

Copy this command to CMD, PowerShell, or the terminal, and click enter.

Warning!: Please use this command on CMD, PowerShell, or the terminal on a folder, or else, the repository will be cloned to the system32, or the current user folder you are using inside the users folder. Because CMD and PowerShell set the default folder as your current user folder you are using inside the users folder, or the system32 folder depending on opening CMD, or PowerShell, normally, or as an admin (A.K.A administrator). So if you wan't to clone this repository, then follow these steps:

  1. Open CMD, PowerShell, or the terminal on a folder you wan't to clone the repository
  2. Go to the top of the folder location, for example:

    3. In this example, my mouse is pointing at something.

  3. Click it with your mouse
  4. Type in "CMD"
  5. And then, click enter.

    6. Now, this will open a CMD window

  6. Copy this command from the "Update: I have just realized that you can just clone the repository, and if you don't know, follow these steps:" section, or copy this command: (it's the same command from the first "Update" section)
git clone https://github.com/eltus/test-website

Now, you have the cloned repository on the folder, you wan't the repository to be cloned, instead of the current user folder you are using inside the users folder, or the system32 folder.

Update: Here's how to install the vscode extensions from part 1, and 3!

  1. Open vscode, or install vscode if you haven't already
  2. Look to the left sidebar, for example:

    3. In this example, my mouse is on the left sidebar.

  3. Click the 4 grouped squares icon (A.K.A the extensions icon), under the 4 other icons (A.K.A the explorer, search, source control, and run and debug icons).
  4. Search those 11 extensions from part 3 of "How to install" section
  5. Click on those 11 extensions. (Or 1 if you are searching them one-by-one)
  6. Then click on the blue "install" button.

And now you have those 11 extensions installed. You can also install those 11 extensions on the vscode extension marketplace, and here's how you do it:

  1. Open your browser, and search "vscode extension marketplace"
  2. Click the first link that appeared
  3. Repeat the same steps from "Update: Here's how to install the vscode extensions from part 1, and 3!" section. (Step 4, 5, and 6, but the blue install button becomes the big green install button! The only thing that is changed are the size, and color)
  4. When you click the big green "install" button, the website says that the extension will be opened on vscode, but it needs your permission to open it, click yes, then it will open the extension and install it (I think)

And now, you have those 11 extension installed from vscode itself, and the vscode extension marketplace.

Update: I just realized that you need git installed in order for the "git clone" command to work, so here's how you install git:

  1. Open your browser, and search, "git download" and click the first link that appeared, or click this link: Git download
  2. Pick your OS (A.K.A Operating System). (It's either Windows, MacOS, or Linux)
  3. Double click the installer
  4. Customize the installation as much as you like. (Not the entire installer tbh)
  5. Click the install button, once you have customize the installation (Not the entire installer tbh, again)

Now, you have git installed, and be able to use the "git clone" command.
